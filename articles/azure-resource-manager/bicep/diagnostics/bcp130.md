# Bicep diagnostic code - BCP130

This diagnostic occurs when [`decorators`](../parameters.md#use-decorators) (such as `@description`, `@allowed`, etc.) are used in contexts where they aren't supportedâ€”specifically within `.bicepparam` files. While decorators remain valid in `.bicep` files for parameter declarations, they have no effect in parameter files and are now explicitly disallowed.

## Description

Decorators are not allowed here.

## Level

Error

## Solutions

Remove decorators from Bicep parameter files. For more information about using parameters, see [Using parameters](../parameters.md#use-decorators).

## Examples

The following example raises the diagnostic because the `@description` decorator is used in a `.bicepparam` file.

```bicep
using './main.bicep'

@description('The name of the storage account to create.')
param storageName = 'mystorageacct'
param storageSKU = 'Standard_LRS'
```

You can fix the diagnostic by removing the `@description` decorator.  

```bicep
using './main.bicep'

param storageName = 'mystorageacct'
param storageSKU = 'Standard_LRS'
```
